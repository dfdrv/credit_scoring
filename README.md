## Описание

Проект реализует пайплайн кредитного скоринга: от загрузки и версионирования данных до обучения ML-моделей, антифрод-проверок и вынесения решения по кредитной заявке (одобрение/отказ).

Основные компоненты:

- **Антифрод-модуль** — выявление подозрительных заявок до передачи в скоринговую модель.
- **Модуль моделирования** — обучение и оценка моделей кредитного скоринга.
- **Скоринговый модуль** — расчёт скоринг-балла и принятие решения по заявке.
- **Пайплайны и скрипты** — автоматизация регулярных процессов (подготовка данных, переобучение, отчётность).
- **Ноутбуки** — исследовательский анализ данных (EDA) и эксперименты с моделями.

## Архитектура и структура проекта

```
credit_scoring/
├── notebooks/            # Jupyter-ноутбуки: EDA, эксперименты, анализ моделей
├── pipeline/              # Пайплайны обработки данных и обучения моделей
├── scripts/               # Вспомогательные и служебные скрипты
├── src/
│   ├── app/
│   │   ├── antifraud/     # Логика антифрод-проверок заявок
│   │   ├── core/          # Общее ядро приложения (базовые сущности, утилиты)
│   │   ├── modelling/     # Обучение, валидация и оценка моделей
│   │   └── scoring/       # Расчёт скоринга и принятие решений
│   └── config/             # Конфигурации приложения и окружений
├── pyproject.toml          # Зависимости и метаданные проекта (Poetry)
├── poetry.lock             # Зафиксированные версии зависимостей
├── setup.cfg                # Конфигурация линтеров/инструментов
├── .dvcignore               # Исключения для DVC
└── CONTRIBUTING.md          # Правила внесения изменений
```

## Стек технологий

- **Python** 3.12+
- **Poetry** — управление зависимостями и виртуальным окружением
- **DVC** — версионирование данных и ML-пайплайнов
- **pandas / numpy / pyarrow** — обработка и анализ данных
- **psycopg2** — работа с PostgreSQL
- **Jupyter** — исследовательские ноутбуки
- **pytest / pytest-cov** — тестирование и покрытие кода
- **wemake-python-styleguide / isort** — статический анализ и форматирование кода

## Установка

Требуется Python версии 3.12 или выше и установленный [Poetry](https://python-poetry.org/docs/#installation).

```bash
git clone https://github.com/dfdrv/credit_scoring.git
cd credit_scoring

# Установка зависимостей (включая dev-зависимости)
poetry install

# Активация виртуального окружения
poetry shell
```

## Работа с данными (DVC)

Данные и артефакты моделей версионируются с помощью [DVC](https://dvc.org/). Перед запуском пайплайнов синхронизируйте данные из удалённого хранилища:

```bash
dvc pull
```

После изменения данных или переобучения моделей зафиксируйте изменения:

```bash
dvc add <путь-к-данным>
dvc push
git add <путь-к-данным>.dvc
git commit -m "Update dataset"
```

## Запуск

Запуск скоринговых и антифрод-пайплайнов осуществляется через скрипты в директории `scripts/` и `pipeline/`. Пример:

```bash
poetry run python scripts/<имя_скрипта>.py
```

Для исследовательской работы с данными и моделями используйте Jupyter:

```bash
poetry run jupyter lab notebooks/
```

> Уточните точные имена входных скриптов и параметры запуска по мере развития проекта — этот раздел стоит дополнить конкретными командами для актуальных пайплайнов.

## Тестирование

```bash
poetry run pytest
```

Отчёт о покрытии кода тестами:

```bash
poetry run pytest --cov=src
```

## Качество кода

Проект использует `wemake-python-styleguide` для статического анализа и `isort` для сортировки импортов:

```bash
poetry run flake8 src
poetry run isort src --check-only
```
